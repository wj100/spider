配置eslint

package.json中的script加上以下代码就可以使用npm run xxx 来执行后面的脚本

"scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "lint": "eslint src",
    "lintInit": "eslint --init"
  },

然后执行npm run lintInit,就会创建一个eslint配置文件,会有一些交互式的问题

  1. 校验程度(检查语法，问题，代码风格)

  1. 模块规范
? How would you like to use ESLint? … 
 To check syntax only

…省略。。。最后会建好一个eslintrc文件

// .eslintrc.js
module.exports = {
   'env': {//环境
   	'browser': true,
   	'es2021': true
   },
   'extends': 'eslint:recommended',//规则
   'parserOptions': {//解析器配置
   //emcaVersion用来指定你想要使用的 ECMAScript 版本
   	'ecmaVersion': 12,
   	// 设置为 "script" (默认)或"module"（如果你的代码是 ECMAScript 模块)
   	'sourceType': 'module',
   	"ecmaFeature":{ //想使用额外的语言特性
   		"jsx":true, //启用jsx
   		"globalReturn":true, //在全局作用域下使用return语句
   		"impliedStrict":true, //启用全局strict mode
   		"experimentalObjectRestSpread":false 
   	}
   },
   //规则
   'rules': {
   	'indent': [
   		'error',
   		'tab'
   	],
   	'linebreak-style': [
   		'error',
   		'unix'
   	],
   	'quotes': [
   		'error',
   		'single'
   	],
   	'semi': [
   		'error',
   		'always'
   	]
   }
}

如果此文件的第一行报错，证明你的eslint已经开始检查了，不要慌，在env中加一个’node’:true

再运行npm run lint即可自动修复一些简单的错误

这种定义未使用的错误只能手动修改

基本使用就是如此，下面说下规则

'extends': 'eslint:recommended'

这是官方的默认规则，rules配置自己的规则，优先级更高

"off"或者0    //关闭规则关闭
"warn"或者1    //在打开的规则作为警告（不影响退出代码）
"error"或者2    //把规则作为一个错误（退出代码触发时为1

一些风格是这样配置的,如：单双引号，结尾分号

rules: {
    "规则名": [规则值, 规则配置]
}
'quotes': [
			'error',
			'single'
		],

规避错误

有时候不想看到编辑器的报错，又暂时不想改，就需要用到避开校验的注释

  • 整个文件不校验
    在文件最顶部写上/* eslint-disable */
  • 对某个代码块不校验
    使用/* eslint-disable */和/* eslint-enable */包起来
  • 对指定的行不校验
console.log(1) // eslint-disable-line;

// eslint-disable-next-line
console.log（1）
